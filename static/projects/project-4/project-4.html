Project Description

This project is the culmination of my Bachelor's degree, representing the synthesis of all the skills I have acquired. Below are the challenges faced during the implementation of the project and the actions taken to address them. If you want to take a look at the code, please follow the GitHub link:

1. Definition of Use Case

Situation: Two use cases were identified - free-flowing crowd and static crowd - each requiring a different approach for density calculation.

Task: Implement a system for density calculation that is suitable for both free-flowing and static crowds.

Action: Choose a thermal imaging approach, where free-flowing crowds are represented as small dots, making density calculation more effective.

Result: A distinction between free-flowing and static crowds was made based on the thermal image characteristics, providing a solution for efficient density calculation.

Reflection: The chosen approach aligns with the visual representation of different crowd types in thermal images, optimizing the system for accurate density calculation in various scenarios.

2. Thermal Camera

Situation: The current thermal camera, Megger TC3231, is incompatible with edge devices and has a narrow field of view.

Task: Find a thermal camera solution compatible with Jetson Nano, offering a wider field of view.

Action: Shortlist FLIR Lepton 3 and FLIR BOSON, which are Jetson Nano compatible and have a wider field of view at 92Â°.

Result: Identified alternative thermal cameras that meet compatibility and field of view requirements for the application.

Reflection: The chosen cameras address the limitations of the current camera, ensuring compatibility and improved coverage for the intended application.

3. Database

Situation: Deploying databases on edge devices is challenging, requiring a reliable solution for data storage and retrieval.

Task: Find a suitable database solution that is compatible with edge devices and easy to deploy.

Action: Choose SQLite as it is available with Python 3 packages and eliminates the need for additional database services.

Result: SQLite is selected as the database solution for data storage and retrieval during thermal detection.

Reflection: The choice of SQLite aligns with the need for a lightweight and easy-to-deploy database solution for edge devices.

4. Jetson Setup

Situation: The code developed on a laptop needs to be set up on Jetson Nano with specific software versions and packages.

Task: Ensure the correct setup of Jetson Nano with the required software versions and packages.

Action: Install dependencies using the pip3 command on the terminal, including lxml, tqdm, absl-py, matplotlib, easydict, and pillow.

Result: Successful installation of dependencies on Jetson Nano using the pip3 command.

Reflection: The chosen approach of installing dependencies via pip3 ensures compatibility and correct version installations for Jetson Nano.

5. TensorFlow on Edge

Situation: Jetpack 4.5 on Jetson Nano does not come with TensorFlow installed, but the code on the laptop uses TensorFlow 2.3 for deep learning.

Task: Install TensorFlow 2.x on Jetson Nano for deep learning compatibility.

Action: Install TensorFlow 2.x on Jetson Nano using the specified command.

Result: Successful installation of TensorFlow 2.x on Jetson Nano.

Reflection: The installation of TensorFlow 2.x ensures compatibility between the code developed on the laptop and the deep learning framework on Jetson Nano.

6. Model Inference and Setup on Jetson Nano

Situation: YOLO v4 model needs to be implemented on Jetson Nano, and the process requires sufficient RAM.

Task: Overcome RAM limitations during the YOLO v4 model inference and setup on Jetson Nano.

Action: Increase the swap memory of the Jetson Nano device from 2 GB to 4 GB to address RAM limitations.

Result: Successful completion of the YOLO v4 model inference and setup on Jetson Nano.

Reflection: Increasing swap memory addresses the RAM constraints of Jetson Nano, enabling the successful implementation of the YOLO v4 model.

7. Algorithm Implementation (YOLO v4)

Situation: YOLO v4 is chosen for human detection with improvements in mean average precision (mAP) and frames per second over YOLO v3.

Task: Implement YOLO v4 algorithm architecture for human detection.

Action: Use CSPDarknet53 as the backbone, incorporate a modified Path aggregation network, a modified spatial attention module, and a modified spatial pyramid pooling in the neck, and utilize dense prediction for locating bounding boxes and classification.

Result: Successful implementation of YOLO v4 with improved accuracy and speed in human detection.

Reflection: The chosen modifications in the YOLO v4 architecture contribute to enhanced accuracy and efficiency in human detection.

8. Algorithm Training (YOLO v4)

Situation: YOLO v4 needs to be trained on a custom dataset to achieve better accuracy in human detection.

Task: Train YOLO v4 on a custom dataset containing thermal and infrared images of crowded environments.

Action: Use transfer learning, annotate the dataset using Roboflow tool, and customize YOLO v4 config file for the specific dataset.

Result: YOLO v4 is successfully trained on the custom dataset, enhancing accuracy in human detection.

Reflection: Transfer learning on a custom dataset provides the algorithm with diverse scenarios, improving its performance in crowded environments.
